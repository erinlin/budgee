## ADDED Requirements

### Requirement: 首頁匯入旅行入口
首頁 SHALL 提供「匯入旅行」按鈕，讓使用者無需進入任何旅行頁面即可匯入 JSON 檔案。

#### Scenario: 首頁觸發匯入
- **WHEN** 使用者在首頁點擊「匯入旅行」按鈕
- **THEN** 系統開啟檔案選擇器，允許選擇 `.json` 檔案

#### Scenario: 匯入成功後導向旅行頁
- **WHEN** 使用者選擇有效的 Budgee JSON 檔案且無衝突（或衝突已解決）
- **THEN** 系統匯入旅行並自動導向該旅行的詳細頁面

### Requirement: 匯入衝突版本比對
當匯入旅行 id 與本地現有旅行重複時，系統 SHALL 在衝突提示中顯示兩個版本的版本資訊，讓使用者有足夠資訊做決定。

#### Scenario: 衝突時顯示版本資訊
- **WHEN** 匯入的旅行 id 與本地旅行重複
- **THEN** 系統顯示衝突提示，內容包含：
  - 本地版本的匯出時間（`exportedAt`）與費用筆數
  - 匯入版本的匯出時間（`exportedAt`）與費用筆數

#### Scenario: 衝突解決：覆蓋
- **WHEN** 使用者在衝突提示中選擇「覆蓋現有旅行」
- **THEN** 系統以匯入資料完整替換本地旅行（包含所有花費與收款）

#### Scenario: 衝突解決：並存
- **WHEN** 使用者在衝突提示中選擇「建立副本」
- **THEN** 系統以新 id 建立旅行副本，標題加上「（匯入副本）」後綴
