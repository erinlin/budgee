## MODIFIED Requirements

### Requirement: 每人餘額計算
系統 SHALL 即時計算每位成員的餘額，分為一般餘額與公費餘額：

一般餘額（排除公費類型花費）：
```
餘額 = 分攤總額 - 代墊總額 - 已收款總額
正值 = 成員仍需繳交給會計
負值 = 會計需退還給成員
```

公費餘額：
```
公費餘額 = 公費支出分攤總額 - 預繳總額
正值 = 需補繳
負值 = 可退費
```

#### Scenario: 正確計算餘額
- **WHEN** 使用者查看收款頁
- **THEN** 每位成員顯示正確的待收/待付金額，並標示顏色（待收 = 警示色、待退 = 正向色）

#### Scenario: 代墊金額自動扣抵
- **WHEN** 某成員有代墊花費
- **THEN** 代墊金額自動從其待收金額中扣除

#### Scenario: 公費餘額獨立於一般餘額
- **WHEN** A 的一般分攤 1000、代墊 0、已收 0，預繳公費 500，公費支出分攤 400
- **THEN** A 的一般餘額為 1000，公費餘額為 -100（可退 100）

---

### Requirement: 收款總覽表格
收款頁 SHALL 以表格呈現所有成員的財務摘要。當旅程中有預收公費紀錄時，表格欄位為：成員、分攤、公費、代墊、已收、餘額(公費)。無預收公費時維持原有欄位：成員、分攤、代墊、已收、餘額。

#### Scenario: 查看收款總覽（含公費）
- **WHEN** 使用者進入收款頁，旅程中有預收公費紀錄
- **THEN** 顯示包含公費欄位的財務摘要表格，餘額欄以「餘額(公費餘額)」格式顯示

#### Scenario: 查看收款總覽（無公費）
- **WHEN** 使用者進入收款頁，旅程中無預收公費紀錄
- **THEN** 顯示原有的財務摘要表格，不含公費欄位

#### Scenario: 公費欄位顯示預繳金額
- **WHEN** A 有兩筆預收公費（1000 + 500），B 有一筆（1000）
- **THEN** A 的公費欄顯示 1500，B 的公費欄顯示 1000

## ADDED Requirements

### Requirement: 收款備註自動帶入公費資訊
當成員有預繳公費時，新增收款紀錄的備註欄 SHALL 自動帶入「含公費 {公費金額}」文字，方便會計辨識收款中包含的公費金額。

#### Scenario: 收款備註自動填入公費
- **WHEN** A 的預繳公費為 500，會計新增 A 的收款紀錄
- **THEN** 備註欄自動預填「含公費 500」，會計可修改

#### Scenario: 無預繳公費時不帶入
- **WHEN** C 沒有預繳公費紀錄，會計新增 C 的收款紀錄
- **THEN** 備註欄維持空白，不帶入公費資訊
